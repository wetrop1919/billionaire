## 🎯 Обзор

**Миллиардер** — многопользовательская экономическая игра с клиент-серверной архитектурой. Игроки перемещаются по сказочному полю, покупают собственность, строят дома и отели, собирают арендную плату и стремятся стать самым богатым.

Игра поддерживает:
- До 8 игроков в одной комнате
- Наблюдателей, следящих за игрой
- AI-ботов для заполнения пустых мест
- Настраиваемые правила
- Сохранение и восстановление после сбоев
- Запись и воспроизведение игр (replay)

## ✨ Возможности

### 🏠 Комнаты
- Создание публичных и приватных комнат
- Защита паролем
- Настройка параметров игры
- Список активных комнат с фильтрацией

### 🎮 Игровой процесс
- 40 клеток игрового поля в сказочной тематике
- 22 улицы в 8 цветовых группах
- 4 железнодорожные станции
- 2 коммунальных предприятия
- 32 карточки «Шанс» и «Фонд»
- Аукционы, торговля, строительство
- Тюрьма и специальная клетка «Веранда»

### 🔄 Сетевая игра
- Автоматическое переподключение при обрыве связи
- Таймер хода (настраиваемый)
- Heartbeat для проверки соединения
- Сохранение состояния игры при отключении

### 🛡 Безопасность
- SSL/TLS шифрование
- HMAC-подпись пакетов
- Argon2id хеширование паролей
- Защита от replay-атак (sequence numbers)
- Rate limiting
- JWT-подобная система токенов

### 🤖 AI
- Боты с настраиваемым уровнем агрессивности
- Разные стратегии покупки и торговли
- Автоматическая замена отключившихся игроков

### 📊 Статистика
- Профиль игрока с подробной статистикой
- История завершённых игр
- Таблица лидеров
- Запись и воспроизведение игр

### Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/wetrop1919/billionaire.git
cd billionaire_game

# Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# Установите зависимости
pip install -r requirements.txt

# Настройте окружение
cp .env.example .env
# Отредактируйте .env — укажите пароли и ключи

# Примените миграции БД
alembic upgrade head
Запуск сервера
bash
# Сгенерируйте SSL-сертификаты (если нет)
mkdir -p certs
openssl req -x509 -newkey rsa:2048 -keyout certs/server.key \
  -out certs/server.crt -days 365 -nodes \
  -subj "/CN=localhost"

# Запустите сервер
python server/main.py
Сервер запустится на 0.0.0.0:8443 (SSL).

Запуск клиента
bash
python client/main.py
Подключитесь к серверу, войдите или зарегистрируйтесь, создайте комнату и начните игру!

Docker
bash
# Запуск через docker-compose
cd deploy
docker-compose up -d

# Проверка статуса
docker-compose ps

# Остановка
docker-compose down
systemd (Linux)
bash
# Скопируйте сервис
sudo cp deploy/billionaire-server.service /etc/systemd/system/

# Активируйте и запустите
sudo systemctl daemon-reload

sudo systemctl start billionaire-server

Особые механики

Веранда — отдельная клетка вне поля, выход стоит 50$
Аукцион — при отказе от покупки, старт 50% от цены
Торговля — обмен собственностью и карточками, деньги в долг (0-50%)
Залог — +10% к стоимости выкупа
Строительство — не требуется монополия на цвет

📖 Полные правила

🌐 Сетевой протокол
Бинарный протокол с JSON-нагрузкой:

text
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  Magic   │ Version  │  Type    │  Flags   │ Payload  │Sequence  │
│  4 bytes │  6 bytes │ 2 bytes  │ 2 bytes  │ 4 bytes  │ 8 bytes  │
├──────────┴──────────┴──────────┴──────────┴──────────┴──────────┤
│                      Payload (JSON / zlib)                      │
├─────────────────────────────────────────────────────────────────┤
│                   HMAC-SHA256 (32 bytes)                        │
└─────────────────────────────────────────────────────────────────┘
56 типов пакетов (AUTH, ROOM, GAME, CHAT, SYSTEM, ADMIN)

HMAC-подпись каждой сессии

Sequence numbers (защита от replay-атак)

Zlib-сжатие для пакетов > 512 байт

📖 Спецификация протокола | API справочник

🛡 Безопасность

Механизм				Описание

SSL/TLS					Шифрование транспортного уровня (TLS 1.2+)
HMAC-SHA256				Подпись каждого пакета сессионным ключом
Argon2id				Хеширование паролей (memory-hard, 64 MB)
JWT-подобные	 		токены	Access (1 час) + Refresh (30 дней)
Sequence Numbers		Защита от replay-атак
Rate Limiting			50 пакетов/сек, 3 сообщения чата/сек
Валидация на сервере	Клиент не доверенный — все проверки на сервере
Защита от zip-бомб		Ограничение размера распакованных данных (10 MB)

🔧 Разработка
Ключевые принципы

SOLID — каждый класс имеет одну ответственность
DRY — общий код в shared/, нет дублирования
KISS — простые решения, избегание избыточной сложности
PEP 8 — строгое соблюдение стиля
Type Hints — полная типизация всего кода
Immutable by default — @dataclass(slots=True, frozen=True)
No magic numbers — все константы в shared/constants.py

Архитектурные паттерны
Паттерн	Где применяется
MVVM	Клиент (PySide6)
Repository	Доступ к БД
Dependency Injection	Контейнеры server/container.py, client/container.py
EventBus	Связь между модулями сервера
State Machine	Игровой движок
Observer	Наблюдатели комнат
Strategy	AI-боты
Builder	Сборка сетевых пакетов
Chain of Responsibility	Security Middleware
Система разрешений

python
